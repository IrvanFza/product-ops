# Getting Help

Product-Ops is an open-source project maintained in limited time. Here's how to
get help efficiently.

## Where to ask

| Question type | Where |
|---|---|
| **Bug** (something is broken) | [GitHub Issues](https://github.com/IrvanFza/product-ops/issues) |
| **Feature idea** | [GitHub Issues](https://github.com/IrvanFza/product-ops/issues) or [Discussions](https://github.com/IrvanFza/product-ops/discussions) |
| **How do I…?** | [GitHub Discussions](https://github.com/IrvanFza/product-ops/discussions) |
| **Setup help** | Run `node doctor.mjs`, then ask in [Discussions](https://github.com/IrvanFza/product-ops/discussions) |
| **Security vulnerability** | See [SECURITY.md](SECURITY.md) — do **not** open a public issue |

## Before opening an issue

1. Search existing issues — someone may have reported it already.
2. Run `node doctor.mjs` — it catches most setup problems.
3. Include your OS, Node.js version, and which AI CLI you're using (Claude
   Code, OpenCode, Codex, Qwen, …).

## What NOT to use GitHub Issues for

- General business or marketing advice.
- Requests to evaluate your specific product idea for you.
- Support for modified forks or unofficial distributions.

These will be closed and redirected to
[Discussions](https://github.com/IrvanFza/product-ops/discussions).
