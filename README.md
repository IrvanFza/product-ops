# Product-Ops

> From idea to operating business — an AI pipeline for solo founders.
> Ideation → brand → business plan (pricing) → marketing → regular operations.

AI-powered, CLI-agnostic **product-creation** pipeline. Paste an idea or a
competitor URL → get market research, a product-fit evaluation, a brand guide,
a business/pricing plan, a marketing plan, and a tracked pipeline — all
generated by whichever AI coding CLI you open inside this folder.

- **Local-first.** Runs on your machine against your files. No server, no account.
- **AI-agnostic.** The logic lives in `modes/*.md`; any agent-skill-standard CLI
  drives it (Claude Code, OpenCode, Codex, Qwen, Copilot, Antigravity, Grok).
- **Human-in-the-loop.** It drafts and evaluates; you decide and click. It never
  auto-publishes, auto-spends, or auto-launches.

Architecture forked from [`santifer/career-ops`](https://github.com/santifer/career-ops).
Skill layer built on [`coreyhaines31/marketingskills`](https://github.com/coreyhaines31/marketingskills)
(45 agent-skills, `product-marketing` foundation).

## Quick start

```bash
npx product-ops init          # scaffold a workspace (or: npx IrvanFza/product-ops init)
# then open your AI CLI inside the folder and paste an idea or competitor URL
```

## Testing

```bash
node test-all.mjs      # run the full test suite (no framework — node:assert/strict)
```

Tests live as `*.test.mjs` at the repo root and cover liveness classification,
the tracker parser, the TSV→`data/products.md` merge, and status
canonicalization. Every test is self-running: `node <file>.test.mjs` exits `0`
on pass, `1` on fail. CI runs the whole suite on every push and pull request.

## Contributing

We work **issue-first**: open an [issue](https://github.com/IrvanFza/product-ops/issues)
to discuss a feature or architecture change *before* writing code (bug fixes,
docs, and new scanner providers can go straight to a PR). When in doubt,
start in [Discussions](https://github.com/IrvanFza/product-ops/discussions).

1. Open an issue (or join a Discussion).
2. Fork and branch (`git checkout -b feature/my-feature`).
3. Make your change; keep it minimal and local-first.
4. `node test-all.mjs` — all tests must pass.
5. Open a PR referencing the issue (`Closes #123`).
6. CI green → review → merge.

See [CONTRIBUTING.md](CONTRIBUTING.md) for the full guide.

## Layout

| Path | Purpose |
|---|---|
| `modes/` | The "brain" — Markdown prompts the AI reads (`_shared.md`, `auto-pipeline.md`, stage modes) |
| `providers/` | Zero-token market-research board modules (ProductHunt, G2, Reddit, Trends…) |
| `*.mjs` | Zero-token Node tools (scan, liveness, generate-pdf, tracker, self-update, doctor) |
| `templates/` | HTML/LaTeX/PDF templates + `states.yml` (lifecycle) + example configs |
| `scaffolder/` | The public `npx product-ops init` bootstrap (separate npm package) |
| `dashboard/` | Optional Go TUI over `data/products.md` |
| `AGENTS.md` | Canonical agent instructions (imported by all CLI entry files) |
| `DATA_CONTRACT.md` | System vs user file boundary (the updater never touches user files) |
